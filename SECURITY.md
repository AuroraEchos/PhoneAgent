# Security Policy

## Supported versions

Security fixes are currently applied to the latest public release and the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for vulnerabilities involving command execution, credential exposure, unsafe confirmation bypass, trajectory leakage, or destructive device actions.

Report the issue privately to the repository maintainer through GitHub's private vulnerability reporting feature when enabled. Include:

- affected commit or version;
- reproduction steps;
- expected and actual behavior;
- security impact;
- a minimal proof of concept with all personal data and credentials removed.

## Operational safety

PhoneAgent controls a real Android device. Run it only on devices and accounts you are authorized to operate. Keep human confirmation enabled for sensitive actions and review trajectories before sharing them.
