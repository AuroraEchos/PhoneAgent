# GitHub Public Release Guide

This repository is prepared for the first public release, `v0.1.0`.

## Before replacing the private repository working tree

Back up the existing private repository and rotate any API key that has been shared outside
your machine. Keep the local `.env`, but never add it to Git.

## Replace the working tree

From the parent directory of the private repository:

```bash
cp -a PhoneAgent PhoneAgent-private-backup
```

Inside the repository, keep only `.git` and the local `.env`, remove the old tracked working
files, then copy the clean release tree into the repository. The exact commands are included
in the accompanying ChatGPT instructions.

## Generate dependencies and validate

```bash
rm -f uv.lock
uv lock
uv sync --extra dev
bash scripts/release_check.sh
```

## Publish a clean public history

The former private history contained development artifacts that should not become public.
Create a new orphan branch for the first public release:

```bash
git checkout --orphan public-release
git rm -rf --cached .
git add .
git commit -m "release: publish PhoneAgent v0.1.0"
git branch -M main
git push --force-with-lease origin main
```

Review the remote repository before changing its visibility.

## Create the tag and GitHub Release

```bash
git tag -a v0.1.0 -m "PhoneAgent v0.1.0"
git push origin v0.1.0
```

Create a GitHub Release from the tag, using `RELEASE_NOTES_v0.1.0.md` as the release body.
Attach the wheel, source distribution, and checksum file generated by the release check.

## Enable GitHub Pages

Open **Settings → Pages**, select **GitHub Actions** as the source, and run the
`Deploy GitHub Pages` workflow. The project site is configured for:

```text
https://auroraechos.github.io/PhoneAgent/
```
